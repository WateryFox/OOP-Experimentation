I developed this project to explore the more advanced features of Object Oriented Programming (OOP) in Python. The goal was to build a combat engine that doesn't just manage health and damage, but also implements strict data privacy and structural hierarchy. By using access modifiers and inheritance, I created a system where different "Hero" archetypes can share basic behaviors while maintaining their own unique attributes. This project served as a deep dive into how professional software manages complex object relationships and keeps data secure.

A key focus of this script was mastering data visibility. I implemented Public, Protected, and Private variables to control how information is accessed. For instance, the hero's gender is a private variable (__gender), meaning it cannot be modified directly from outside the class. This forced me to write specific "Getter" and "Setter" methods, ensuring that any changes to the character's data are handled safely and intentionally. This approach is essential for preventing bugs in larger projects where multiple parts of the code might try to change the same variable at once.

To make the system more scalable, I implemented the HeroTanker class using Inheritance. By having the Tanker class inherit from the base Hero class, I was able to reuse all the existing battle logic without rewriting it. I used the super() function to trigger the parent class's initialization while overriding specific stats to give the Tanker higher health and defense. This demonstrates a "DRY" (Don't Repeat Yourself) coding philosophy, showing how a single base class can be used to generate dozens of unique character variations like Mages, Assassins, or Healers.

I also integrated Class Variables (like jumlah and move) to keep track of the global game state, such as how many heroes exist and how many turns have passed in total. The battle loop uses a simple but effective damage formula that factors in the defender's stats, creating a more strategic feel than a simple health subtraction. I used the time module to add pauses between turns, transforming a simple console script into a more dramatic, readable simulation. This project has given me a strong foundation in designing systems that are both organized and extensible.
